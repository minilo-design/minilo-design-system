# Minilo Design System Documentation

[minilo.io](https://minilo.io)

## Vision

Our vision is to **redefine the bridge between design and development** by creating a **streamlined, niche design system** that eliminates inefficiencies, reduces friction, and accelerates the journey from **concept to code**.

## **Purpose Statement**

The purpose of this design system is to:

1. **Standardize Design Patterns:** Ensure consistency across all UI components.
2. **Accelerate Development:** Reduce repetitive tasks with pre-built, reusable components.
3. **Enhance Collaboration:** Enable seamless communication between designers and developers.
4. **Minimize Ambiguity:** Provide clear documentation and usage guidelines.
5. **Focus on Niche Design Needs:** Address specific design challenges unique to our targeted audience.

## **Core Goals**

1. **Efficiency:** Reduce the design-to-development cycle time by providing ready-to-use assets and components.
2. **Scalability:** Create a modular system with small and large projects.
3. **Accessibility:** Ensure all components follow accessibility best practices (e.g., WCAG).
4. **Consistency:** Maintain a unified design language across platforms and devices.
5. **Innovation:** Continuously evolve based on user feedback and industry trends.

## **Core Principles**

- **Minimalism:** Simple yet functional design components.
- **Accessibility:** Inclusive and user-friendly designs.
- **Responsiveness:** Adapt seamlessly across devices.
- **Customizability:** Flexible components tailored to unique requirements.
- **Transparency:** Clear documentation and open-source contributions.

## **Value Proposition**

- **For Designers:** A consistent design library that reduces guesswork and promotes creativity.
- **For Developers:** Clean, reusable components with clear documentation and examples.
- **For Businesses:** Faster time-to-market and reduced development costs.

## Design Principles

1. **Border Radius**: Use a fixed border-radius of 8px or 8% of the rectangle size, whichever is more consistent with the design.
2. **Border Color**: All borders should have a fixed color: `#E9E9E9`.
3. **Container Background**: The background color of all containers should be white for a clean and minimalistic look.
4. **Text Styling**: Titles or text inside containers or buttons should always be bold to ensure clarity and emphasis.
5. **Layout Background**: The layout background color should be `#FDFCFF` to maintain a consistent, soft aesthetic.
6. All components follow a unified visual language.
7. The framework can adapt to new features easily.
8. WCAG-compliant components.

### Typography

1. Font family:
   - Noto Sans is the default
   - Fallback to sans serif
2. Mobile Font Sizes (scaled down from desktop):
   - H1: 28px (was 32px)
   - H2: 22px (was 24px)

### Spacing & Layout Guidelines

**1. Grid System**

- **Base Spacing Unit**: 8px
  - This will act as the fundamental unit of spacing in your design system. All spacing (margins, padding, etc.) will be based on multiples of this unit (e.g., 8px, 16px, 24px).
  - **Consistency**: Using a consistent spacing unit will ensure a visually balanced and harmonious layout across different platforms and devices.

**2. Responsive Layout & Breakpoints**

A responsive layout ensures that your design adapts seamlessly across various screen sizes. Here's a breakdown of the breakpoints:

- **Extra Small (Mobile Portrait)**: `max-width: 480px`
  - This is for small mobile devices, with the design adjusted for smaller screens.
  - Adjustments: Single-column layouts, larger touch targets.
- **Small (Mobile Landscape / Tablets)**: `min-width: 481px` and `max-width: 768px`
  - This is for medium-sized devices like tablets and landscape mobile screens.
  - Adjustments: Consider adding more space for touch elements and flexible grid usage.
- **Medium (Tablets / Small Laptops)**: `min-width: 769px` and `max-width: 1024px`
  - For tablets and smaller laptop screens.
  - Adjustments: Two-column layouts or larger typography for better readability.
- **Large (Desktop / Large Laptops)**: `min-width: 1025px` and `max-width: 1440px`
  - For typical desktop screen sizes.
  - Adjustments: Grid systems for multi-column layouts, fixed-width containers, and more complex content areas.
- **Extra Large (Wide Screens)**: `min-width: 1441px`
  - For larger desktops and wide screens.
  - Adjustments: Maximize content width, and use flexible grids to ensure content remains balanced.

**3. Layout Grid**

- **Columns**:
  - **Mobile**: 1 column
  - **Tablet**: 2 columns
  - **Desktop**: 12 columns (with a flexible gutter for spacing between columns)
- **Gutters**:
  - **Mobile and Tablet**: 16px (for tighter layouts on smaller screens)
  - **Desktop**: 24px (wider gutters for larger screens)

**4. Margins & Padding**

- **Container Padding**:
  - **Mobile**: 16px
  - **Tablet & Desktop**: 24px
- **Outer Margin**:
  - **Mobile**: 8px
  - **Tablet & Desktop**: 16px

**5. Typography**

- **Line Height**: 1.4x the font size for readability.
- **Text Spacing**: Use the base spacing unit (8px) for consistent spacing between text elements like paragraphs and headings.

## Design Tokens

### **Design Tokens**

Design tokens are a set of variables that define the visual design aspects of your system. These tokens allow you to consistently apply styles across different platforms and devices while making it easier to manage and update your design system. Below are examples of how to define your design tokens:

### **1. Colors**

Colors define the primary visual elements of your design.

- **Primary Color**:
  - `color-primary: #FFFFFF` (Light mode - White)
  - `color-primary-dark: #000000` (Dark mode - Black)
- **Secondary Color**:
  - `color-secondary: #0061A8` (Blue for accents, buttons)
  - `color-secondary-light: #B3D7F2` (Light Blue for hover states or secondary buttons)
- **Neutral Color**:
  - `color-neutral-1: #FDFCFF` (Soft background color for layouts)
  - `color-neutral-2: #E9E9E9` (Border and divider color)
  - `color-text-primary: #333333` (Primary text color)
  - `color-text-secondary: #999999` (Secondary text color for less emphasized elements)
- **Accent Color**:
  - `color-accent: #E63946` (Red for alerts, warnings, or badges)
  - `color-accent-light: #FFD3D5` (Light red for hover or focus states)

### **2. Typography**

Typography defines the font properties like size, weight, and line height. These tokens allow consistent text styling across all elements.

- **Font Sizes**:
  - `font-size-xs: 12px` (Small text, such as captions)
  - `font-size-sm: 14px` (Body text)
  - `font-size-md: 16px` (Regular text or titles)
  - `font-size-lg: 20px` (Headings or larger text)
  - `font-size-xl: 24px` (Main headings, larger titles)
- **Font Weights**:
  - `font-weight-regular: 400` (Normal weight for body text)
  - `font-weight-bold: 700` (Bold weight for headings or important text)
- **Line Heights**:
  - `line-height-xs: 1.2` (Tighter line height for small text)
  - `line-height-sm: 1.4` (Default line height for body text)
  - `line-height-md: 1.5` (For larger text or headings)

### **3. Spacing**

Spacing defines the margins, padding, and gaps between elements in your design.

- **Small Spacing**:
  - `spacing-xs: 4px` (Small margins/padding for tightly packed elements)
- **Medium Spacing**:
  - `spacing-sm: 8px` (Standard space between elements, such as icons and text)
  - `spacing-md: 16px` (Padding for containers or large gaps between sections)
- **Large Spacing**:
  - `spacing-lg: 24px` (Wide margins or padding for larger elements or sections)
  - `spacing-xl: 32px` (Spacing for very large containers or page sections)

### **4. Border Radius**

Defines the rounding of the edges of UI elements.

- `border-radius-sm: 4px` (Slightly rounded corners for small elements)
- `border-radius-md: 8px` (Standard rounded corners for most elements)
- `border-radius-lg: 16px` (Large rounded corners for prominent elements)

### **5. Shadows**

Defines the depth effect of elements, although your framework might avoid shadows based on your initial request.

- `shadow-none: none` (No shadow, to maintain minimalism)
- `shadow-sm: 0px 1px 2px rgba(0, 0, 0, 0.1)` (Very light shadow for subtle depth)
- `shadow-md: 0px 2px 4px rgba(0, 0, 0, 0.2)` (Moderate shadow for noticeable depth)

## Atomic Components

**Loading State (with Shimmer Loading)**

In modern UIs, loading states are crucial for providing visual feedback while content is being loaded. One of the popular loading states is the **Shimmer Loading Effect**, which mimics a wave of light over content to indicate that the content is loading.

**Shimmer Loading:**

- **Shimmer Effect**: A continuous animation that creates the illusion of light sweeping across elements to indicate loading.
- **Default Loading State**: An animated shimmer effect over components like buttons, text, or cards, giving users the impression that content is loading.

**1. Buttons – Loading State**

Incorporating a loading state for buttons is essential when an action is processing (e.g., form submission).

- **Loading State**:
  - `background-color: #E0E0E0;` (Light gray background to indicate the button is disabled)
  - `color: transparent;` (Hide text or icon to prevent interaction)
  - `border: 1px solid #BDBDBD;` (Subtle border color)
  - **Shimmer Animation**:
    - Apply a shimmer animation to the button’s background to indicate activity.
    - Example:
      ```css
      @keyframes shimmer {
        0% {
          background-position: -1000px 0;
        }
        100% {
          background-position: 1000px 0;
        }
      }
      .button-loading {
        animation: shimmer 2s infinite linear;
        background: linear-gradient(
          90deg,
          #f0f0f0 25%,
          #e0e0e0 50%,
          #f0f0f0 75%
        );
        background-size: 200% 100%;
        position: relative;
      }
      ```

**2. Inputs – Loading State**

Inputs can show a loading state, especially in forms or search bars that require fetching results from an API.

- **Loading State**:
  - `background-color: #F0F0F0;` (Subtle background color)
  - `border-color: #BDBDBD;` (Gray border)
  - **Shimmer Animation**:
    ```css
    .input-loading {
      animation: shimmer 1.5s infinite linear;
      background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
      background-size: 200% 100%;
      position: relative;
    }
    ```

**3. Cards / Containers – Loading State**

For content-heavy pages or cards, a loading state with the shimmer effect is useful.

- **Loading State**:
  - `background-color: #F0F0F0;` (Light background to simulate an empty state)
  - **Shimmer Animation**:
    ```css
    .card-loading {
      animation: shimmer 2s infinite linear;
      background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
      background-size: 200% 100%;
      position: relative;
      height: 150px; /* or height of the card */
      border-radius: border-radius-md;
    }
    ```

**4. General Shimmer Animation**

Here’s the CSS code for creating the shimmer effect that can be applied to any loading state component:

```css
@keyframes shimmer {
  0% {
    background-position: -1000px 0;
  }
  100% {
    background-position: 1000px 0;
  }
}

.shimmer-loading {
  animation: shimmer 2s infinite linear;
  background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  position: relative;
}
```

**5. Loading State in Atomic Components**

- **Buttons**:
  - **Normal**: Default button with text.
  - **Loading**: Display shimmer animation in place of text and change the button to show it’s in progress.
- **Inputs**:
  - **Normal**: Regular text input.
  - **Loading**: Display the shimmer effect on the input field as the content is being processed or fetched.
- **Cards / Containers**:
  - **Normal**: Display content (e.g., images, text).
  - **Loading**: Use shimmer animation to show a placeholder content area while data is being fetched.

**Best Practices for Implementing Loading State**

- **Visibility**: Always ensure the loading state is clearly visible to the user (the shimmer effect should be prominent enough to be noticed).
- **Duration**: Keep loading times to a minimum to avoid frustrating users. Use loading indicators only when necessary (e.g., when waiting for API calls or data fetching).
- **Fallback**: For long-loading operations, consider adding a text label like "Loading..." or an icon to let users know what is happening.

### **Progress to Molecules and Organisms**

After you've established your **atomic components**, the next step is to combine these atoms into more complex structures called **molecules** and **organisms**. These elements are higher-level components that allow you to build more complex UI elements with reusable pieces from your atomic components.

**1. Molecules**

Molecules are combinations of two or more atomic components that work together as a unit but are still relatively simple. These elements are reusable across your design and are essential building blocks for creating organisms.

**1.1. Card Component (Molecule)**

A **card** is a commonly used UI element that typically contains a heading, image, body text, and action buttons. Cards can be created by combining atoms like buttons, text, and images.

**Structure of Card Component**:

- **Card Container**:
  - `background-color: color-neutral-1;`
  - `border-radius: border-radius-md;`
  - `box-shadow: none;` (to keep the design minimal)
  - `padding: spacing-md;`
- **Card Image**:
  - `width: 100%;`
  - `height: auto;`
  - `border-radius: border-radius-sm;`
- **Card Text**:
  - `font-size: font-size-sm;`
  - `color: color-text-primary;`
- **Card Button**:
  - Combine the button atomic component (with the default button styles).
  - Add a **loading state** as defined earlier when content is being loaded in the card.

**Card Component Example**:

```html
<div class="card">
  <!-- Ellipse -->
  <div class="ellipse"></div>

  <!-- Headline -->
  <h3 class="card-headline">Headline</h3>

  <!-- Supporting Text -->
  <p class="card-supporting-text">
    Some supporting text goes here to describe the card.
  </p>

  <!-- Apply Now Button -->
  <button class="apply-now">Text Button</button>
</div>
```

An **input group** combines a text input with additional elements like buttons, icons, or labels. Input groups are useful when there are multiple related actions, such as a search bar with a button or an input with a validation message.

**Structure of Input Group**:

- **Input Field**: Atomic input component.
- **Button/Icon**: Atomic button or icon component (e.g., a search icon button inside the input field).

**Input Group Example**:

**1.3. Modal (Molecule)**

A **modal** is a common UI pattern used for displaying content over the current page. It’s typically composed of a header, body content, and actions like buttons.

**Structure of Modal**:

- **Modal Background**: A semi-transparent background that blurs or darkens the content behind it.
- **Modal Content**: Contains the main content of the modal (text, forms, etc.).
- **Modal Actions**: Buttons are used to close or confirm the modal’s action (use the button atomic component for actions).

**Modal Example**:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Modal Example</title>
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <!-- Button to trigger modal -->
    <button class="open-modal-btn">Open Modal</button>

    <!-- Modal Overlay and Modal Content -->
    <div class="modal-overlay">
      <div class="modal">
        <div class="modal-header">
          <h2 class="modal-title">Modal Title</h2>
        </div>
        <div class="modal-body">
          <p>Some content or form goes here...</p>
        </div>
        <div class="modal-footer">
          <button class="modal-btn-close">Close</button>
          <button class="modal-btn-confirm">Confirm</button>
        </div>
      </div>
    </div>

    <script src="script.js"></script>
  </body>
</html>
```

```css
// Get modal elements
const modalOverlay = document.querySelector('.modal-overlay');
const openModalBtn = document.querySelector('.open-modal-btn');
const closeModalBtn = document.querySelector('.modal-btn-close');

// Open modal when button is clicked
openModalBtn.addEventListener('click', () => {

// Close modal when close button is clicked
closeModalBtn.addEventListener('click', () => {
    modalOverlay.style.display = 'none'; // Hide the modal
});

// Close modal if clicked outside the modal box
window.addEventListener('click', (event) => {
    if (event.target === modalOverlay) {
        modalOverlay.style.display = 'none'; // Hide the modal
    }
});
```

**2. Organisms**

Organisms are more complex UI components that are made up of multiple molecules and atoms. They represent larger sections or groups of content that perform specific functions on the page. Organisms are typically used to build out entire layouts or functional areas like navigation bars, footers, and headers.

**2.1. Navigation Bar (Organism)**

A **navigation bar** (or **navbar**) is typically located at the top of the page and provides links to the main sections of the site or app. It can contain a combination of logos, links, dropdowns, and user action buttons.

**Structure of Navigation Bar**:

- **Logo**: Typically a logo or brand name.
- **Nav Links**: Links to different sections (e.g., Home, About, Services).
- **Search Bar**: A combined input group that allows users to search.
- **User Actions**: Buttons for login, profile, or notifications.

**Navigation Bar Example**:

```html
<nav class="navbar">
  <div class="navbar-content">
    <!-- Logo Section -->
    <div class="navbar-logo">
      <img
        src="https://via.placeholder.com/28"
        alt="Logo"
        class="navbar-logo-img"
      />
    </div>

    <!-- Navigation Links -->
    <div class="navbar-links">
      <a href="#" class="nav-link selected">Home</a>
      <a href="#" class="nav-link">About</a>
      <a href="#" class="nav-link">Services</a>
      <a href="#" class="nav-link">Contact</a>
    </div>
  </div>
</nav>
```

### **Document Animations for Modal Open/Close Effects**

Use CSS keyframes and transitions to create smooth animations for modal windows.

Example: Modal Open/Close Animation

```css
/* Modal Overlay */
.modal-overlay {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background: rgba(0, 0, 0, 0.5);
  display: flex;
  justify-content: center;
  align-items: center;
  opacity: 0;
  visibility: hidden;
  transition: opacity 0.3s ease, visibility 0.3s ease;
}

/* Modal Content */
.modal {
  background: #ffffff;
  padding: 20px;
  border-radius: 8px;
  transform: translateY(-20px);
  transition: transform 0.3s ease, opacity 0.3s ease;
  opacity: 0;
}

/* Open Modal State */
.modal-overlay.active {
  opacity: 1;
  visibility: visible;
}

.modal-overlay.active .modal {
  transform: translateY(0);
  opacity: 1;
}
```

JavaScript to Trigger Modal

```jsx
const modalOverlay = document.querySelector(".modal-overlay");
const openModalBtn = document.querySelector("#openModal");
const closeModalBtn = document.querySelector("#closeModal");

openModalBtn.addEventListener("click", () => {
  modalOverlay.classList.add("active");
});

closeModalBtn.addEventListener("click", () => {
  modalOverlay.classList.remove("active");
});
```

### **Shimmer Loading Animation**

A shimmer effect for loading states enhances the visual appeal during content load.

Example: Shimmer Effect

```css
/* Shimmer Background */
.shimmer {
  width: 100%;
  height: 20px;
  background: linear-gradient(90deg, #f0f0f0 25%, #e9e9e9 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
  border-radius: 4px;
}

/* Keyframes */
@keyframes shimmer {
  0% {
    background-position: -100% 0;
  }
  100% {
    background-position: 100% 0;
  }
}
```

Usage in HTML

```html
<div class="shimmer" style="width: 300px; height: 20px;"></div>
<div
  class="shimmer"
  style="width: 200px; height: 20px; margin-top: 10px;"
></div>
```
